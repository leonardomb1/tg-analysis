  ---
  Sugestões para pivotar para Análise de Dados em Segurança do Trabalho

  1. Usar bases de dados públicas já existentes

  Em vez de coletar dados via protótipo IoT (que é custoso e demorado para um
  TCC), você pode trabalhar com dados reais:
  - CATWEB/SmartLab do MPT — dados de acidentes de trabalho no Brasil (CATs
  registradas)
  - RAIS/CAGED — dados de emprego por setor, permitindo cruzar com taxas de
  acidentes
  - DATASUS/SUS — internações e óbitos por causas externas relacionadas a
  trabalho
  - OIT/ILO — dados internacionais para comparação

  2. Possíveis perguntas de pesquisa (análise de dados)

  - Quais setores industriais têm maior incidência de acidentes e como isso
  evoluiu nos últimos 10 anos?
  - Existe correlação entre porte da empresa e taxa de acidentes?
  - Quais são os padrões sazonais/temporais de acidentes de trabalho?
  - É possível criar um modelo preditivo de risco de acidentes com base em
  variáveis como setor, região, turno, tipo de atividade?
  - Qual o impacto econômico (dias perdidos, custos previdenciários) dos
  acidentes por setor?

  3. Metodologia orientada a dados

  - Análise exploratória (EDA) — estatísticas descritivas, visualizações,
  identificação de padrões
  - Análise de séries temporais — tendências e sazonalidade de acidentes
  - Modelos preditivos — regressão logística, árvores de decisão ou random
  forest para classificar risco
  - Clusterização — agrupar setores/regiões por perfil de risco usando K-means
  ou DBSCAN
  - Dashboards interativos — criar painéis em Python (Streamlit/Dash) ou Power
  BI para visualização

  4. Como aproveitar o que você já tem

  Você não precisa jogar tudo fora. Pode manter o tema IoT + segurança do
  trabalho, mas mudar o foco:
  - Em vez de construir o sistema IoT, analisar dados que sistemas como esses
  gerariam
  - Usar dados simulados ou públicos para demonstrar como a análise de dados
  melhora a tomada de decisão em segurança
  - O protótipo Arduino/RFID poderia ser uma pequena prova de conceito que gera
  dados para análise, não o produto principal

  5. Proposta reformulada (exemplo)

  Título: "Análise de dados de acidentes de trabalho em ambientes industriais e 
  logísticos: identificação de padrões e modelo preditivo de risco"

  Objetivo: Aplicar técnicas de análise de dados e aprendizado de máquina sobre 
  bases públicas de acidentes de trabalho para identificar fatores de risco, 
  padrões temporais e construir um modelo preditivo que auxilie gestores de 
  segurança na tomada de decisão preventiva.

  6. Ferramentas sugeridas

  - Python (pandas, scikit-learn, matplotlib, seaborn)
  - Jupyter Notebooks para documentação reprodutível
  - Streamlit ou Dash para dashboard interativo
  - PostgreSQL para armazenamento (já está no seu escopo)
